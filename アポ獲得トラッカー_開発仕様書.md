# アポ獲得トラッカー 開発仕様書(Claude Code引き継ぎ用)

## 背景・目的
新卒研修中の営業アポ取りにおいて、新卒メンバーの行動量(架電数・アポ獲得数)を
チームで共有・可視化するための管理ツール。既存版はClaude Artifacts(HTML単体+
window.storage)で作成していたが、複数人が同時に書き込むと保存エラーが頻発するため、
Claude Codeで本格的なデータベースを使った版に作り直したい。

## 想定ユーザー
- 新卒5〜6名(架電・アポ獲得の実績を各自で入力)
- 研修担当者(全体の進捗を管理・閲覧)
- 全員がスマホまたはPCのブラウザからアクセスする(専用アプリ不要)

## 推奨スタック
- フロントエンド: React + Vite(シンプルな構成でよい。Next.jsでも可)
- データベース: Supabase(PostgreSQL、無料枠あり、リアルタイム更新にも対応)
- ホスティング: Vercel(GitHub連携で自動デプロイ、無料枠で十分)
- 認証: 今回は不要(社内ツールのため、リンクを知っている人が使う想定でよい)

## データモデル(Supabaseのテーブル設計イメージ)

### reps(担当者)
| カラム | 型 | 説明 |
|---|---|---|
| id | uuid (PK) | |
| name | text | 担当者名 |
| month_goal | int | 月間アポ獲得目標(初期値20、担当者ごとに個別設定可) |
| week_goal | int | 週間アポ獲得目標(初期値5、担当者ごとに個別設定可) |
| created_at | timestamp | |

### daily_logs(日次記録)
| カラム | 型 | 説明 |
|---|---|---|
| id | uuid (PK) | |
| rep_id | uuid (FK -> reps.id) | |
| date | date | 記録日 |
| calls | int | 架電数 |
| appointments | int | アポ獲得数 |
| created_at | timestamp | |

### call_plans(週次の架電予定数)
| カラム | 型 | 説明 |
|---|---|---|
| id | uuid (PK) | |
| rep_id | uuid (FK -> reps.id) | |
| week_start | date | その週の開始日(月曜) |
| planned_calls | int | その週の架電予定数 |

補足: 月目標・週目標は上記のように reps テーブルに持たせるシンプルな設計でも良いし、
月ごとに変える場合は月ごとの目標テーブル(rep_id, year, month, goal)に分離してもよい。
現行版では「福島さんだけ月目標40件」のように個人ごとに異なる値を設定できる必要がある。

## 画面構成・機能一覧(既存のHTML版から引き継ぐ機能)

### 1. ヘッダー
- タイトル「アポ獲得トラッカー」
- 月切り替えボタン(‹ 2026年7月 ›)で表示月を前後に移動できる
- 表示月を変えると、下記の月間サマリー・週次振り返りの両方が連動して切り替わる

### 2. チーム月間サマリー
- チーム合計アポ獲得数 / チーム合計目標(全員の月目標の合計)
- 進捗バー
- 架電数合計、アポ獲得率(アポ獲得数 ÷ 架電数)

### 3. 担当者カード(グリッド表示、人数分)
- 担当者名(編集可能。変更すると過去の記録・目標もすべて新しい名前に紐づく)
- 順位(アポ獲得数の多い順)
- 架電数、アポ獲得数(表示中の月の合計)
- アポ獲得率
- 月目標に対する進捗バー
- 月目標の数値(担当者ごとにクリックして編集可能)

### 4. メンバー管理
- 新しいメンバーを名前入力して追加できる

### 5. 週次振り返り
- 週は「月曜〜金曜」の5日間で区切る。**月をまたがないこと**が重要な要件
  (例:月末が水曜で終わる場合、その週は月〜水の3日間だけの範囲として扱う)
- その月の全週があらかじめ選択肢として表示される(記録の有無に関わらず)
- 週ごとに以下を担当者別に一覧表示:
  - 架電予定数(その週にクリックして入力・編集できる)
  - 架電数(実績)
  - 予定達成率(架電数 ÷ 架電予定数)
  - アポ獲得数
  - アポ獲得週目標(担当者ごとにクリックして編集可能)
  - アポ獲得率(アポ獲得数 ÷ 架電数)
- チーム週間アポ獲得数 / チーム週間目標(全員の週目標の合計)の進捗バー

### 6. 実績記録フォーム
- 担当者(プルダウン)、日付、架電数、アポ獲得数を入力して記録
- 記録一覧(直近の記録を日付降順で表示、削除も可能)

## デザインの方向性
- ダークテーマ(電話交換台・営業コールセンターをイメージした配色)
- ベース背景: 濃紺〜チャコール系(#12151C 相当)
- アクセントカラー: 架電数=ブルー系、アポ獲得数=アンバー/ゴールド系
- 等幅フォント(数字表示)+ ゴシック体(日本語UI)の組み合わせ
- 詳細な配色・フォントは既存HTML版のCSSを参照可能(添付ファイル参照)

## 補足・注意点
- 既存のHTML版(`アポ獲得トラッカー.html`)は機能の完成形として参考になるので、
  Claude Codeとの会話に一緒に添付することを推奨する。
- 複数人が同時に同じデータ(週次の架電予定数など)を編集する可能性があるため、
  Supabase側でのUPDATE処理は「最後に保存した内容で上書き」で問題ない
  (厳密な排他制御は不要な想定)。
- 将来的に認証(担当者ごとのログイン)を追加したくなった場合は、
  Supabase Authを使うと拡張しやすい。
